# Experience Rubric

## Questions

- Can the user see the next useful action within 5 seconds?
- Does the page explain state through data and controls rather than instructions?
- Does each recommendation show why it exists and whether it is safe to apply?
- Can the user recover from interruption, low energy, or failed plan without losing trust?
- Are empty, loading, error, and unauthorized states usable?
- Does mobile/narrow layout keep controls readable and non-overlapping?
- Are destructive or high-risk actions visibly gated?
- Does the UI distinguish facts, suggestions, drafts, and applied changes?

## Core Moments

- First login and seed state.
- Morning plan.
- Focus session start/end.
- Low-conversion closeout.
- Evening review.
- Weekly/monthly report.
- Recovery mode.
- Simulation exam result.
- Version update check.

## Evidence

Use screenshots or browser observations for visual claims. Use API output for data claims. Use product docs for intent claims.
