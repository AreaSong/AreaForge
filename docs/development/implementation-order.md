# 开发顺序

## 当前状态

已经完成：

- monorepo 基础。
- Next.js Web 应用。
- Prisma schema。
- Docker Compose 基础。
- 首页作战台雏形。
- 专注计时器前端状态。
- 核心规则包基础。
- 文档拆分。

## 下一步主线

1. 单管理员登录。
2. 数据库 seed：初始化科目和管理员。
3. 今日作战台读取真实数据。
4. 每日任务 CRUD。
5. 学习计时持久化。
6. 打卡与连续天数。
7. 每晚复盘保存。
8. 考纲树基础版。
9. 笔记和附件上传。
10. AI 鞭策建议接入。

## 第一阶段验收

达到以下条件才算进入真正 MVP：

- 登录后才能访问作战台。
- 可以创建任务。
- 可以开始、暂停、结束一次计时。
- 计时记录保存到数据库。
- 今日统计来自数据库。
- 可以提交晚间复盘。
- `pnpm check` 通过。
