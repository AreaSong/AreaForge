# 产品上手（Getting Started）

## 定位

本文面向第一次使用 AreaForge 的人，讲"应用已经跑起来之后，怎么开始用它备考"。

- 如果你还没有把应用跑起来：本地开发环境见 [开发设置](../development/setup.md)，服务器自托管部署见 [自托管操作者上手](../deployment/operator-onboarding.md)。
- 各页面的完整用法见 [使用指南](user-guide.md)，环境变量说明见 [配置参考](configuration.md)，常见问题见 [FAQ 与排障](faq.md)。

## 第一次登录

AreaForge 是单管理员私有应用，没有注册入口。管理员账号在初始化时由环境变量写入数据库：

1. 在 `.env` 中设置 `AUTH_ADMIN_EMAIL`，并用 `pnpm auth:hash '<你的密码>'` 生成密码哈希写入 `AUTH_ADMIN_PASSWORD_HASH`。
2. 运行 `pnpm db:seed` 初始化管理员和七门基础科目（数学、英语、政治、408 四门）。
3. 打开应用地址，进入 `/login`，用上面的邮箱和密码登录。

登录后进入今日作战台（首页 `/`），这是每天打开应用的第一屏。

## 前 30 分钟建议路径

按下面顺序走一遍，学习闭环的所有环节就都接触到了：

1. **认识作战台**：首页展示双节点倒计时、今日任务、学习时长、连续打卡、风险等级、阶段称号和鞭策文案。第一天大部分是空的，正常。
2. **初始化考纲**（`/syllabus`）：为主要科目建立大纲树，支持 Markdown 导入章节结构。考纲是"学到了哪里"的落点，任务、笔记、错题之后都会关联到节点上。
3. **建今天的第一个任务**：回到首页任务区，选科目、写任务标题、选任务类型、估计时长，可以关联到考纲节点。
4. **开始一次专注计时**：点开始后计时器进入专注模式，页面聚焦当前科目和任务；支持暂停、继续、结束。刷新页面不会丢失进行中的计时。
5. **认真做结束收口**：结束计时时需要回答学习质量、是否有效学习、理解程度、最小产出和下一步动作。这是反假学习检查的入口，打卡也以"完成一段计时并提交收口"为准，只打开应用不算打卡。
6. **留下笔记或错题**：把这段学习的产出记进 `/notes`（支持 PDF、PNG、JPEG、WebP 附件）或 `/mistakes`，并关联考纲节点。
7. **晚上做复盘**：在首页复盘区回答今天学了多久、完成了什么、哪里失控了、明天最小必须完成什么。启用 AI 后可以生成复盘建议和明日任务草稿，但草稿必须你确认后才生效。

## 之后每天怎么用

```text
早上看作战台（推荐任务 + 欠账 + 风险） -> 白天专注计时 + 结构化收口 -> 随手记笔记/错题 -> 晚上复盘 -> 按建议调整明天
```

坚持一到两周后，`/analytics` 的趋势、`/reports` 的周审判和考纲作战地图才开始有分析价值。长期机制（任务债务、恢复模式、掌握证明、全真模拟、阶段调整）的用法见 [使用指南](user-guide.md)。

## 需要提前知道的边界

- 所有数据存在你自己的 PostgreSQL 和上传目录里，没有云端同步。
- AI 只在显式触发时调用，只生成建议或草稿，不会覆盖你的记录；动机档案和完整情绪记录默认不发给 AI。
- 未完成的任务不会消失，会进入任务债务池等待处理。
- 应用内的版本中心（`/settings`）只提交受控更新请求，真正的更新由服务器侧执行。
