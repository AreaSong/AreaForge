# 使用指南（User Guide）

## 定位

本文面向 AreaForge 的使用者，按"每天怎么用"和"长期怎么用"讲清各页面和机制。它只写怎么用；模块的设计规则见 `docs/modules/**`，第一次上手见 [产品上手](getting-started.md)。

## 页面总览

| 路径 | 名称 | 用途 |
|---|---|---|
| `/` | 今日作战台 | 每天的第一屏：倒计时、今日任务、专注计时、复盘、风险与鞭策 |
| `/syllabus` | 考纲进度 | 大纲树、节点状态、作战地图、掌握证明 |
| `/notes` | 笔记资料 | 文本笔记与私有附件，按科目/节点/掌握状态/复习提醒筛选 |
| `/mistakes` | 错题与掌握证明 | 错题记录、错因、复习提醒 |
| `/analytics` | 数据统计 | 学习时长、完成率、连续性、风险项趋势 |
| `/reports` | 周期报告 | 周审判与月复盘，含确认/驳回决策 |
| `/simulation` | 全真模拟 | 结构化模拟考试、阶段计划与调整草稿 |
| `/motivation` | 动机封存 | 封存开始的原因，关键节点唤醒 |
| `/settings` | 设置 | 账号信息、运行状态、版本中心与更新策略 |

界面为电脑优先设计，移动端响应式适配；深浅色与状态主题随学习状态变化。

## 每日闭环

### 早上：看作战台

首页按优先级组织信息：第一优先级是当前推荐任务、开始计时和风险等级；其次是今日指标、欠账任务和考纲概览；最后是错题提醒、复盘提醒和阶段称号。连续高完成时会突出连胜，断签后会突出欠账和恢复动作。

### 白天：任务与专注计时

- 任务带科目、类型、优先级、预计时长，可关联考纲节点；状态在待开始、进行中、已完成、已延期、已放弃之间流转。
- 点"开始计时"进入专注模式：计时器放大，显示当前科目、任务和节点，其余信息弱化。支持暂停/继续；同一时间只允许一个进行中的计时，刷新页面后会自动恢复。
- 结束计时必须做结构化收口：学习质量评分、是否有效学习、理解程度、最小产出、下一步动作。"有效学习"的判断会进入打卡、统计和风险计算。
- 只看课没输出、只抄笔记没理解、计时很长但任务没推进，会被标记为低转化学习并纳入周报统计——这是反假学习机制，别绕它。

### 随手：笔记与错题

- 笔记支持文本和附件（PDF、PNG、JPEG、WebP），附件保存在私有目录并通过鉴权接口访问，列表不暴露磁盘路径。
- 每份资料建议写清属于哪个节点、解决什么问题、下次何时再看；上传文件不等于掌握。
- 错题记录错因和复习提醒时间，到期错题会出现在作战台提醒和统计风险里。

### 晚上：复盘

复盘回答五个问题：今天学了多久、完成了什么、哪里失控了、最该保留的一个动作是什么、明天最小必须完成什么。保存后固化当日打卡快照。启用 AI 后可生成复盘建议和明日任务草稿；AI 不会覆盖你的原始复盘。

## 长期机制

### 打卡与连续性

打卡不等于打开应用，必须完成至少一次有效学习动作（一段计时 + 收口）。连续打卡、断签次数、连续低效天数会影响风险等级、鞭策语气和阶段称号。

### 任务债务

未完成任务自动进入欠账池，按影响分为可接受、需要补、影响阶段、计划崩坏四级。处理方式有补做、延期、拆小、合并、放弃、改成复习任务。系统会生成重排建议，但从不自动应用：你在首页任务区勾选建议项后，显式确认、驳回或应用所选。

### 考纲进度与掌握证明

- 节点状态覆盖未开始到掌握、薄弱、放弃暂缓；支持打勾、打叉、星标、警告标记。
- 作战地图聚合所有科目进度，标出薄弱、遗忘风险、错题高发区域并给出优先处理节点和下一步动作。
- 把节点标为"掌握"需要证据：可设置掌握条件（看完课、写过理解、做过题、错题复盘、延迟复测），并引用同节点下的任务、计时、笔记、错题或复测记录作为显式证据。证据不足时系统会拒绝写入掌握状态——打勾只是状态，掌握需要证明。
- 复测记录 `passed/failed/partial`；只有通过计入延迟复测证据，失败只留历史，不自动降级。

### 恢复模式

连续断签、完成率长期过低或你主动点"我需要恢复"时进入恢复模式。作战台聚焦一个最小可执行任务（默认目标 30–90 分钟），不强制补完所有欠账；完成恢复任务后逐步回到正常计划。文案严肃但不羞辱。

### 周审判与月复盘

`/reports` 的周审判汇总一周真实执行（时长、完成率、欠账、低转化、错题、最大短板和下周必须压住的一个问题）；月复盘检查阶段计划是否仍然有效。报告给出策略草稿和下周期决策预览，你可以确认或驳回；确认只冻结快照和草稿，不会自动修改任务或阶段计划。

### 全真模拟与阶段调整

`/simulation` 创建结构化模拟考试，按整场和分科目记录目标分、实际分、用时、空题、失分原因、心态和总结，考后自动生成复盘文本。基于结果可生成阶段调整草稿（本地规则或显式触发 AI），草稿必须显式确认后才会更新阶段计划。

### 阶段称号、鞭策与动机

- 阶段称号（初醒、入炉、淬火、重铸、稳态、破局、冲刺）按连续打卡、时长、完成率等计算，影响首页展示、鞭策语气和动态主题。
- 鞭策引擎按上升期到危险期的真实状态选择语气：学得好继续加压，学得差给最低可执行动作，不灌鸡汤也不只批评。
- `/motivation` 封存你开始的原因和给未来自己的话，只在连续失守、重大复盘、模拟前后等关键节点唤醒，不每天出现。

## AI 边界（使用者视角）

- AI 需要在配置中显式启用（见 [配置参考](configuration.md)），并且只在你显式触发时调用。
- AI 产出的一切都是建议或草稿：复盘建议、明日任务草稿、阶段调整草稿，均需确认后生效。
- 默认不发送给 AI：动机档案、完整情绪记录、完整复盘正文、附件内容、完整任务标题。情绪标签可以进入上下文。
- AI 不可用或失败时自动回退到本地规则，功能不中断。

## 设置与版本中心

`/settings` 显示账号、运行状态和版本更新入口。版本中心只做两件事：查看当前版本与更新通道状态、提交受控更新请求。它不会在网页里执行更新——签名校验、备份、migration、切换和回滚全部由服务器侧完成。更新策略的含义见 [配置参考](configuration.md) 和 [GitHub Release updater](../deployment/github-release-updater.md)。
