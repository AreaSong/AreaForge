# 任务计划

## 目标

任务计划用于管理每天应该完成什么，并和计时、大纲、笔记、错题、复盘关联。

## 字段

- 科目。
- 任务标题。
- 任务类型。
- 预计时长。
- 实际时长。
- 优先级。
- 完成状态。
- 所属大纲节点。
- 完成后的复盘记录。

## 初始科目

- 数学。
- 英语。
- 408 数据结构。
- 408 计算机组成原理。
- 408 操作系统。
- 408 计算机网络。
- 政治。

## 状态

- 待开始。
- 进行中。
- 已完成。
- 已延期。
- 已放弃。

## 规则

- 未完成任务进入任务债务系统。
- 完成任务后可以触发鞭策文案。
- 任务可以关联到学习计时。
- 任务可以关联到大纲节点和笔记。

## 当前行为

- 今日任务表单支持选择任务类型，类型写入 `StudyTask.type`；任务卡片展示类型、状态和预计时长。
- 拆小任务通过 `StudyTask.parentTaskId` 建立父子关系，债务动作同步写 `TaskDebtEvent` 事件账本。
- 重排建议支持确认、驳回和用户所选项应用记录，从不自动应用。
- 批量任务调整或自动阶段联动不在当前范围，需单独确认。

实现进度与批次证据见 [功能追踪矩阵](../development/feature-traceability.md)。
