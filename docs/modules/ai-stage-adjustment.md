# AI 阶段调整建议

## 目标

AI 阶段调整建议用于根据长期数据辅助修正学习计划。

## 输入

- 阶段目标。
- 任务完成率。
- 科目投入占比。
- 错题与复盘趋势。
- 模拟考试结果。
- 连续打卡和断签记录。
- 低转化学习记录。

## 输出

- 当前阶段风险结论。
- 下阶段重点科目。
- 任务强度调整建议。
- 恢复、强化或冲刺模式建议。
- 需要砍掉、延期或重排的任务类型。
- Batch 6 后，本地规则建议可持久化为 `StageAdjustmentDraft`，并关联到 `StagePlan`。
- Package D Batch D3 后，用户可显式生成长期 AI 草稿，成功时持久化为 `StageAdjustmentDraft.source="ai"`。

## 边界

- AI 只能生成建议。
- AI 不能自动覆盖阶段计划。
- 用户确认后才能应用调整。
- AI 输出必须结构化校验。
- Package D Batch D3 已完成长期 AI 草稿显式触发路径；普通页面打开、报告 GET、SSR 和后台任务仍不调用长期 AI。
- D3 只发送最小聚合字段和阶段目标摘要，不发送动机档案、完整情绪记录、完整复盘正文、附件内容、文件路径或完整任务标题。
- Batch 6 的确认应用只更新关联阶段计划和审计记录，不自动任务重排、不批量修改任务。

## 第二阶段定位

长期 AI 阶段调整属于第二阶段增强。第一版只保留 AI 鞭策、复盘建议和明日任务建议；阶段调整在 Package B Batch 6 中完成本地规则草稿持久化和用户确认边界，在 Package D Batch D3 中补齐显式 AI 草稿。Package D Batch D4 已完成长期风险/主题闭环的只读证据链；报告驱动阶段应用不进入当前范围，生产发布已由 Package E / `v0.1.5` 记录独立闭环。
