# 考纲进度树与作战地图

## 目标

把“学了多久”进一步落到“学到了哪里”。

## 考纲树

每个科目有一棵大纲树。

节点层级：

- 科目。
- 章节。
- 知识点。
- 题型或专题。

节点状态：

- 未开始。
- 学习中。
- 已学完。
- 需要复习。
- 掌握。
- 薄弱。
- 放弃或暂缓。

节点标记：

- 打勾：已完成或已掌握。
- 打叉：当前不会、理解失败或需要重学。
- 星标：高频重点或个人重点。
- 警告：长期未复习或错题集中。

## 作战地图

作战地图是所有科目进度的可视化概览。

格子状态：

- 未开始。
- 学习中。
- 已覆盖。
- 已验证。
- 薄弱。
- 遗忘风险。
- 错题高发。

第一版先做树形进度 + 网格概览。

## 第二阶段增强

- 标记长期未复习节点。
- 根据错题集中度生成遗忘风险。
- 根据计时、任务、错题、复盘自动更新节点状态。
- 作战地图支持更强的视觉表达和筛选。

## 当前行为

- `packages/core` 提供 `evaluateSyllabusMapSignal` 单节点规则和 `summarizeSyllabusMap` 聚合规则；`/syllabus` API、页面和组件展示分科聚合摘要、地图状态分布、推荐筛选、优先处理节点、行动类型筛选和节点下一步动作。
- 考纲服务读取已有任务、计时、笔记和错题的最近更新时间，向作战地图和掌握证明传入证据新鲜度，用于提示遗忘风险和证据过旧。
- 聚合规则统计覆盖率、验证率、风险等级，推荐错题高发、薄弱、遗忘风险等筛选项。
- 作战地图风险接入 `GET /api/analytics/long-term-risks` 和同源页面面板。
- 结构化复习历史或用户确认后应用如未来需要，必须另走确认。

实现进度与批次证据见 [功能追踪矩阵](../development/feature-traceability.md)。
