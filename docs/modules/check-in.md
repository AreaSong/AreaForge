# 打卡与连续性

## 目标

打卡系统用于记录用户是否每天回到学习状态，并为风险判断、恢复模式和鞭策文案提供依据。

## 记录内容

- 学习日期。
- 是否完成最低学习动作。
- 当日总学习时长。
- 当日有效学习时长。
- 当日任务完成率。
- 是否完成晚间复盘。

## 连续性指标

- 当前连续打卡天数。
- 历史最长连续打卡天数。
- 断签次数。
- 最近一次断签日期。
- 连续低效学习天数。

## 规则

- 打卡不等于打开应用，必须至少完成一次有效学习动作。
- 第一版可定义为：完成一段计时并提交收口记录。
- 连续断签会提高风险等级。
- 连续断签达到阈值后进入恢复模式。
