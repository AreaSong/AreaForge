# 动态主题

## 目标

AreaForge 的视觉风格根据学习状态变化，让界面本身成为反馈。

## 主题状态

### 正常推进

深色作战台，冷静、克制、稳定。

### 锻造状态

连续高完成时触发，突出连胜、累计投入、阶段突破。

### 警报状态

断签或欠账升高时触发，突出风险和恢复动作。

### 恢复状态

连续失守时触发，降低计时入口任务密度，聚焦最小任务。

### 冲刺状态

临近考试时触发，优先真题、模拟、错题复盘。

## 设计限制

- 不做一味装饰。
- 不用大面积单一色调。
- 不让主题影响可读性。
- 不让动效干扰学习。

## 当前实现

- `packages/core` 根据冲刺窗口、风险状态、连续打卡和任务完成率生成 `themeState`。
- 首页外壳和状态主题面板消费 `themeState`，展示正常推进、锻造、警报、恢复和冲刺五类主题。
- 主题面板只展示真实信号：风险等级、连续打卡、任务完成率、欠账、断签、低转化、阶段压强、倒计时和复盘状态。
- 恢复主题会让计时器聚焦最小任务，并在地图区弱化欠账预览，避免恢复状态下继续制造压迫；完整任务列表仍在任务区可查看。
- 冲刺主题前置倒计时、阶段压强和有效学习结果，保持真题、模拟、错题复盘优先。
- 所有主题都保持深色高对比卡片，主题色只用于徽标、边框和少量强调，不改变正文可读性。
