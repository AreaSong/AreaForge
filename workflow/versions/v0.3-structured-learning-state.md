# v0.3 结构化学习状态

## 目标

把第一版中仍依赖文本备注、实时派生或轻量状态的核心学习行为，升级为可审计、可统计、可回滚的结构化状态层。

本版本是 `v0.2` 完整第一版高风险闭环中的 Package B 专项执行线，不是可绕过的后置增强。v0.2 的第一版验收依赖本版本完成。

当前进度：Package B Batch 0-6 已全部完成；后续结构化状态变化只作为新的高风险确认包推进。

## 范围

- `StudySession` 结构化收口字段。
- `CheckIn` 每日快照。
- 任务债务事件账本和父子任务关系。
- 恢复模式状态、手动触发和退出记录。
- 掌握证明条件、证据引用和复测记录。
- 结构化模拟考试模型已由 Batch 5 接入；阶段计划和阶段调整草稿的基础模型已由 Batch 6 接入。
- 结构化统计口径更新。
- 旧数据兼容和最小回填策略。

## 不包含

- 删除旧字段或不可逆压缩历史数据。
- 自动批量放弃、删除或重排任务。
- 多用户隔离改造。
- 生产 migration deploy 执行。

## 入口条件

- `tasks/backlog/0015-structured-state-migration.md` 完成确认。
- migration 方案明确模型变更、旧数据映射、兼容期、验证和回滚。
- 本地临时库 migration 和 seed 路径可重复验证。

## 验收标准

- 新增结构只做 additive migration，不直接删除旧字段。
- 首页、统计、报告、任务债务、恢复模式和掌握证明读取结构化数据。
- 历史文本备注仍可读，不因新结构丢失。
- `pnpm db:validate`、migration dry run、相关 API/页面烟测和 `pnpm check` 通过。

## 退出条件

- 当前派生口径和结构化口径差异有说明。
- 后续第二阶段能力能直接依赖结构化状态，而不需要继续从文本备注猜测。

## 风险

- 新旧口径并存期间统计不一致。
- 回填不完整导致历史数据看似缺失。
- migration 在生产环境执行前需要备份和回滚方案。
