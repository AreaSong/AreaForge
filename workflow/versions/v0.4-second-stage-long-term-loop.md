# v0.4 第二阶段长期闭环

## 计划状态

- 计划状态：**已完成**。完成证据以 `docs/development/docs-100-completion-record.md` 为准。

## 目标

实现 `docs/product/feature-scope.md` 中第二阶段增强，让 AreaForge 从每日执行工具升级为长期备考调整系统。

## 范围

- 完整全真模拟考试模型和 2026 年 12 月同步自测专题流程。
- 周审判、月复盘从只读报告升级为阶段决策入口。
- 任务债务自动重排建议，但不自动应用。
- 知识点遗忘风险、笔记复习提醒和错题复盘趋势；当前基础版把错题记录更新趋势归入遗忘风险与周期报告，精确复盘次数需结构化事件后实现。
- 作战地图高级筛选和风险可视化。
- 状态主题、阶段称号、动机唤醒和长期压强调节联动。
- AI 长期阶段调整建议，用户确认后才能应用。

## 不包含

- AI 自动覆盖阶段计划。
- AI 自动生成完整长期学习计划。
- 多人排名、公开分享或社交对比。
- 网页内一键部署或服务器命令。

## 入口条件

- v0.2 和 v0.3 的第一版与结构化状态闭环完成。
- Package C 第一版真实 AI provider 的隐私、费用、限流和日志边界已确认；长期阶段 AI 草稿最小字段清单和显式触发入口已由 Package D Batch D3 完成，应用流仍需后续确认。
- Package D Batch D1 已完成报告确认、驳回、冻结快照、确认草稿、审计和只读回放；Batch D2 已完成任务债务重排确认、驳回和所选项应用；Batch D3 已完成长期阶段 AI 草稿；Batch D4 已完成长期风险和主题闭环补强；Batch D5 已完成 Package D 收口。
- 模拟考试 migration 已由 Package B Batch 5 完成；阶段计划和阶段调整草稿的 migration 已由 Package B Batch 6 完成。

## 验收标准

- 可以创建、查看、复盘结构化模拟考试。
- 第一次同步自测能作为阶段重校准输入。
- 周/月报告能生成可确认的下周期策略，不自动改写计划。
- 债务重排、遗忘风险和阶段调整都以建议形式呈现，并保留用户确认边界。
- `pnpm check`、核心规则测试、AI fallback 测试、关键页面烟测通过。

## 退出条件

- 第二阶段增强项在 `docs/development/feature-traceability.md` 中均达到“已完成”或被源事实重新标记为暂缓。
- 长期数据进入 AI 前的最小字段清单、脱敏规则和失败回退路径可审计。
- Package D D1-D5 均完成：D1 报告决策入口、D2 任务债务重排确认流、D3 长期阶段 AI 草稿、D4 长期风险/主题闭环和 D5 证据收口均已完成。
- `pnpm docs:completion` 不再列 Package D、长期阶段 AI 或 Package E blocker；后续功能变更需继续同步 docs/tasks/workflow 和 release 证据。

## 风险

- 长期建议可能被误解为自动计划。
- 模拟结果和阶段调整会影响长期备考节奏，必须用户确认后应用。
- AI 外呼涉及费用、隐私和可用性风险。
