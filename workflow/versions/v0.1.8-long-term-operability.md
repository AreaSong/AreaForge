# v0.1.8 长期治理与并发保护发布候选

## 当前状态

- 当前版本状态：本地 Release candidate；workspace package 已同步为 `0.1.8`。
- 是否已有线上 Release：没有。生产仍运行签名 Release `v0.1.7` / 应用版本 `0.1.7`。
- 证据边界：尚未形成干净 exact-commit CI、`v0.1.8` tag、签名资产或生产 rollout 证据。

## 目标

把 `v0.1.7` 之后已经完成的 OPS-005 Expected-Before V2、OPS-006 业务并发保护、长期运营证据绑定、供应链校验和体验修复固定为一个可审查、可签名、可独立部署的补丁候选。

## Planning Gate

- 目标：形成干净候选提交，并在独立确认后创建与该提交匹配的签名 `v0.1.8` Release。
- 非目标：本计划不授权 tag/Release、生产 backup/migration/deploy、timer/队列变更、controlled-write probe、restore、自动应用策略变化或 residual 关闭。
- Exact docs：`docs/development/release-train.md`、`docs/development/high-risk-confirmation-packets.md`、`docs/development/ops-006-business-state-concurrency-design.md`、`docs/development/update-request-expected-before-design.md`、`docs/development/residual-risk-ledger.md`。
- Owner skill：`areaforge-release-operator`，供应链由 `areaforge-supply-chain` 复核，验证由 `areaforge-validation-driver` 驱动。
- Validation profile：Release candidate full gate。
- Source baseline：生产 `v0.1.7`；候选 exact commit 在干净 checkpoint 后固定。
- Residual risk IDs：`AF-RISK-OPS-001`、`AF-RISK-SC-001`、`AF-RISK-SC-004`、`AF-RISK-OPS-005`、`AF-RISK-OPS-006`、`AF-RISK-UX-001`。
- Release trigger：只有用户以真实 40 位候选提交明确确认签名 `v0.1.8` Release 后才能创建或推送 tag。
- Apply boundary：候选准备为 R0/R1；签名 Release、基础生产 rollout 和 controlled probe 是三个独立确认阶段。
- 回滚：候选阶段只回滚候选提交；未来生产应用失败时优先回滚 Web/agent 到已验证的 `v0.1.7` digest，并保留 additive index、队列和 decision history。

## 范围

- OPS-005 schema V2、snapshot binding、TTL、幂等、原子发布、processing reconciliation、不可变 decision history 和共享 production-state lock。
- OPS-006 partial unique index、task/session CAS、结束计时单次副作用、CheckIn advisory lock、稳定 409 和隔离 PostgreSQL `local_verified`。
- OPS-005/006 生产证据 validator/preflight、独立 rollout/probe confirmation scope、Release/source-at-commit/reconciliation/rollback 交叉绑定。
- Release evidence、供应链 strict parser、evidence-only closeout binding 和只读长期 gate 加固。
- 当前 checkout 的移动端溢出、页面体验、CI Prisma 前置和治理文档同步修复。

## 入口条件

- workspace 版本全部为 `0.1.8`，工作区形成单一干净候选提交。
- `pnpm check`、release/updater/supply-chain/OPS-005/006/治理/文档门禁和 `git diff --check` 全部通过。
- current-bound desktop/mobile UX 与 runtime identity 绑定候选提交。
- 候选提交进入 GitHub 默认分支并取得匹配 CI 后，才能申请签名 Release 确认。

## 验收与退出

- 本地候选完成：版本一致、完整门禁通过、干净 checkpoint、匹配 UX/CI 证据齐备。
- 签名 Release 完成：manifest、SBOM、provenance、`SHA256SUMS`、cosign bundle 和 Web/migration immutable digest 通过 strict 校验。
- 本计划不以生产完成退出；生产 rollout、OPS-006 controlled probe、OPS-001/release evidence 和 residual 人工关账继续按独立确认包执行。

## Release Notes 草案

- 加固版本中心 Expected-Before V2，拒绝过期或状态漂移的更新请求并保留可恢复决策证据。
- 为计时与任务状态增加数据库单活跃约束、CAS 和单次副作用保护。
- 增强签名 Release、供应链、备份/附件 reconciliation 和长期运营证据的严格绑定。
- 修复考纲窄屏溢出、CI Prisma 生成顺序，并统一当前 checkout 的体验与运行身份校验。
