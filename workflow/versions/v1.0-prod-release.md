# v1.0 生产发布闭环

## 目标

把 AreaForge 推进到可部署、可备份、可恢复、可回滚的私有生产版本。

## 范围

- Next.js standalone Docker 镜像。
- PostgreSQL 16 生产部署。
- Nginx HTTPS 反向代理。
- 生产 `docker-compose.prod.yml` 校验。
- 数据库、上传目录、生产 `.env` 和当前版本 tag 备份。
- Prisma migration deploy 流程。
- 临时库和临时上传目录恢复演练。
- 发布前、发布后和失败回滚检查清单。

## 不包含

- 网页内一键更新。
- 网页内执行服务器命令。
- 不经备份的破坏性 migration。
- 公开 SaaS、多用户或复杂权限系统。

## 入口条件

- v0.2 到 v0.4 的产品能力达到目标范围，或明确哪些能力延后。
- `tasks/backlog/0014-deployment-backup-release.md` 的高风险确认包完成。
- `docs/development/production-release-runbook.md` 和 `docs/development/docs-100-acceptance-evidence.md` 已完成当前状态复核。
- 生产密钥、域名、Nginx、数据卷和备份目录准备完成。

## 验收标准

- 生产 PostgreSQL 不暴露公网端口。
- Web 仅绑定本机端口供 Nginx 反代。
- 发布前生成数据库、上传目录、生产 `.env` 和当前版本 tag 备份。
- migration deploy 前存在可回滚备份点。
- 每日 `pg_dump` 和上传目录同周期备份，至少保留 14 天。
- 临时库恢复后可登录、打开首页，并完成附件 metadata/文件本体对账。
- `pnpm check`、`docker compose config`、`docker compose -f docker-compose.prod.yml config` 通过。

## 退出条件

- 生产版本可访问、可登录、核心学习链路可用。
- 恢复演练有记录，失败回滚路径可执行。

## 风险

- 发布中断或 migration 后不可逆。
- 备份缺失导致无法恢复。
- 上传目录和数据库 metadata 不一致。
- 生产变量或密钥泄露。
