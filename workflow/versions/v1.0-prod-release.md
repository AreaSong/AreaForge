# v1.0 生产发布闭环

## 目标

把 AreaForge 推进到可部署、可备份、可恢复、可回滚的私有生产版本。

## 范围

- Next.js standalone Docker 镜像。
- PostgreSQL 16 生产部署。
- Nginx HTTPS 反向代理。
- 生产 `docker-compose.prod.yml` 校验。
- 数据库、上传目录、生产 `.env` 和当前版本 tag 备份。
- Prisma migration deploy 流程。
- 临时库和临时上传目录恢复演练。
- 发布前、发布后和失败回滚检查清单。
- GitHub Release 驱动的服务器侧受控自动更新器。

## 不包含

- Web runtime 直接执行服务器命令的一键更新。
- 网页内执行服务器命令。
- 跳过签名/hash 校验的自动更新。
- 静默 major 更新。
- 不经备份的破坏性 migration。
- 公开 SaaS、多用户或复杂权限系统。

## 入口条件

- v0.2 到 v0.4 的产品能力达到目标范围，或明确哪些能力延后。
- `tasks/backlog/0014-deployment-backup-release.md` 的高风险确认包逐批完成；当前 Package E Batch E1 生产配置与发布工件预检、Batch E2 发布前备份与恢复演练、Batch E3 本机单机生产发布与 migration deploy、Batch E4 本机生产回滚演练与收口均已完成。
- `docs/development/production-release-runbook.md` 和 `docs/development/docs-100-acceptance-evidence.md` 已完成当前状态复核。
- 生产密钥、域名、Nginx、数据卷和备份目录准备完成。

## 验收标准

- 生产 PostgreSQL 不暴露公网端口。
- Web 仅绑定本机端口供 Nginx 反代。
- 发布前生成数据库、上传目录、生产 `.env` 和当前版本 tag 备份。
- migration deploy 前存在可回滚备份点。
- migration deploy 执行载体明确记录为受控 release 工作目录、一次性 migration job 或 `not-applicable`。
- 每日 `pg_dump` 和上传目录同周期备份，至少保留 14 天。
- 临时库恢复后可登录、打开首页，并完成附件 metadata/文件本体对账。
- 发布记录通过 `pnpm release:evidence:validate <release-record.txt> [attachment-reconciliation.csv]`，包含 `envBackupSha256`、compose/Nginx 副本路径、回滚演练字段和只读 `report_only` 附件对账。
- `pnpm check`、`pnpm package-e:preflight`、`docker compose config`、`docker compose --env-file .env.example -f docker-compose.prod.yml config` 通过；裸跑生产 compose 若缺真实生产 env，预期会因必填变量失败。
- GitHub Release updater 通过 `pnpm github-release-updater:preflight`，Release manifest 包含 `webImageDigest`、`migrationImageDigest`、`SHA256SUMS`、`SHA256SUMS.sig` 和 `autoApply` 策略；服务器默认 `AREAFORGE_AUTO_APPLY=none`，patch 自动更新必须显式配置。

## 退出条件

- 生产版本可访问、可登录、核心学习链路可用。
- 恢复演练有记录，失败回滚路径可执行。
- 发布记录能回答镜像 digest、compose/Nginx hash、备份 hash、migration runner、恢复演练、发布后烟测、回滚计划、回滚耗时和残余风险。
- Package E E1-E4 均完成：生产配置预检、发布前备份与恢复演练、本机单机生产发布与 migration deploy、生产回滚演练与收口证据。
- 服务器侧 GitHub Release updater 已有脚本、systemd timer 示例、manifest 合约、Release workflow 和只读验证门禁；真实远端已按 `docs/deployment/github-release-updater.md` 完成 `v0.1.5` 签名 Release 更新，记录见 `docs/development/package-e-remote-github-release-record.md`。
- GitHub Actions 已包含 CI workflow 和 Release workflow：CI 跑 shellcheck、updater preflight、Package E / 风险 / docs 门禁和 `pnpm check`；Release workflow 构建并发布 Web / migration 镜像和 Release assets。
- `pnpm docs:completion` 通过，或若源事实明确延后生产发布，则不得宣称 docs 100%。

## 风险

- 发布中断或 migration 后不可逆。
- 备份缺失导致无法恢复。
- 上传目录和数据库 metadata 不一致。
- 生产变量或密钥泄露。
